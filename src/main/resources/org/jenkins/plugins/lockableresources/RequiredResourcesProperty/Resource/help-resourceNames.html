When a build is scheduled, it will attempt to lock the specified resources. If some (or all) the resources are already locked by another build, the build will be queued until they are released. You can request multiple resources by name using a space separated list (of either resource name or label) or using groovy. It is possible to specify an amount for requested resources below.

If resource name starts with label: then it is treated as a space separated AND list that the resource must have.

If resource name starts with groovy: then it is treated as a groovy expression to be evaluated each time a resource is checked to be appropriate for a build. The expression must result into a boolean value. The following variables are available:

resourceName
as per resource configuration
resourceDescription
as per resource configuration
resourceLabels
java.util.List<String> of labels as per resource configuration

For matrix jobs, axis names and axis values are referencable as well. Examples:

  • groovy:resourceLabels.contains("hardcoded")
  • groovy:resourceLabels.contains(axisName)
  • groovy:resourceName == axisName

Build parameters can be used here also: ${PARAMETER} or %PARAMETER%

Examples:

  • Resource Name [OR]: Test1 Test2
  • Labels (cannot exist as a resource name) [OR]: lab-1 lab-2
  • Labels [AND]: label:sanity 7.2
  • Groovy script: groovy:resourceName == axisName
