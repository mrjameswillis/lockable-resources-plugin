When used (use with EXACT label matching - label:) the lock will look for the most matched resource.
Defaults to global setting when creating a new required resource.

Example setup:

  • Resource1 labels: sanity 2x1 7.2 pdu
  • Resource2 labels: sanity 2x1 7.2

  • Required resource request: label:sanity 2x1 7.2

Resource2 will be selected first because the match is 3/3 while Resource1 is 3/4.
